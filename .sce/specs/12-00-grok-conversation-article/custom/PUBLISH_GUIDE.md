# 发布到 GitHub Discussions 指南

## 步骤 1: 启用 Discussions 功能

1. 打开你的 GitHub 仓库：https://github.com/heguangyong/scene-capability-engine
2. 点击顶部的 **Settings** 标签
3. 在左侧菜单找到 **Features** 部分
4. 勾选 **Discussions** 复选框
5. 点击 **Set up discussions** 按钮

## 步骤 2: 创建新的 Discussion

1. 在仓库页面，点击顶部的 **Discussions** 标签
2. 点击右上角的 **New discussion** 按钮
3. 选择分类（建议选择 **Show and tell** 或 **General**）

## 步骤 3: 填写内容

```

```

## 🎯 核心观点

这是一场关于 AI 发展趋势的深度对话，探讨了：

1. **范式转变**：AI 从"被动投喂"到"主动探寻"，再到"原则驱动引擎"
2. **哲学基础**：心学的"知行合一"与唯识论的"缘起缘灭"为 AI 实践提供理论支撑
3. **实践落地**：kiro-spec-engine 将理念转化为可操作的工程实践
4. **未来展望**：AI 成为信息化世界的中心，人类专家转为"元专家"角色

## 💡 关键洞见

- **逻辑是思维的绳索**：语言即世界，心外无物
- **周期本就是逻辑绳索**：缘起缘灭，无需额外的人类外在绳索
- **偏置才是常态**：死像与本相，活化与转化
- **用 spec-driven 的外在形式，守护内在的无**

## 🚀 实践项目

这些理念在 [kiro-spec-engine](https://github.com/heguangyong/scene-capability-engine) 项目中得到了实践：

- 精确解决 session 上下文吞噬问题
- 文档驱动 + 自动化治理
- 高内聚、低耦合的天然实现
- 原则驱动 Agent 的着力点
- Ultrawork spirit + Sisyphus 隐喻
